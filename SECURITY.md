# Security

## Report a problem privately

Do not open a public issue for leaked keys, exposed reports, authentication
bypasses, or private incident data.

Use GitHub's **Report a vulnerability** button in the repository Security tab.
Include clear steps to reproduce the problem, but include only the minimum safe
data needed.

## Supported version

Security fixes are made on the latest code in the `main` branch.

## What happens next

A maintainer will confirm the report, check its impact, prepare a fix, and share
public details only after sensitive information is safe.
